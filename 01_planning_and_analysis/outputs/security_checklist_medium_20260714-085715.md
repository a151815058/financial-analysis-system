## 🟡 中級新增項目（12 項）

### 構面 1：存取控制 — 新增
- [ ] 項次 5：應依系統功能及權限，區分系統管理者及一般使用者帳號
- [ ] 項次 6：應針對特權帳號（管理者、Root 等）定期清查使用授權
- [ ] 項次 7：使用者於資源存取結束後，系統應強制登出或自動鎖定
- [ ] 項次 8：資通系統若閒置逾時（建議 15 分鐘）應自動登出

### 構面 2：事件日誌與可歸責性 — 新增
- [ ] 項次 18：應針對日誌進行審查，並保留審查紀錄
- [ ] 項次 22：日誌處理失效時，應產生警示訊息
- [ ] 項次 26：應運用雜湊或其他方式確保日誌資訊之完整性
- [ ] 項次 27：定期將日誌備份至原系統外之伺服器

### 構面 3：營運持續計畫 — 新增
- [ ] 項次 30：應定期測試備份資料之還原程序
- [ ] 項次 31：定期執行備份資料還原測試
- [ ] 項次 32：備份資料應異地存放
- [ ] 項次 34：針對核心資通系統建立備援機制
- [ ] 項次 35：定期執行備援系統切換演練

### 構面 4：識別與鑑別 — 新增
- [ ] 項次 37：應使用多因子驗證機制（如自然人憑證、FIDO）
- [ ] 項次 44：使用圖形驗證碼或簡訊/Email OTP 進行身分驗證輔助
- [ ] 項次 45：密碼重設應安全地驗證使用者身分

### 構面 5：系統與服務獲得 — 新增
- [ ] 項次 49：應進行威脅建模（Threat Modeling），識別系統威脅與弱點
- [ ] 項次 50：依威脅建模結果產出安全需求修正
- [ ] 項次 50-1：應建立安全需求追蹤矩陣
- [ ] 項次 54：應針對原始碼進行安全檢測（SAST）
- [ ] 項次 55：應禁止於軟體元件中使用已知弱點之版本
- [ ] 項次 57：執行「滲透測試」安全檢測
- [ ] 項次 59：應管理系統組態，確保安全設定（關閉不必要服務/埠口）
- [ ] 項次 60：應建立軟體元件清單（SBOM），管理版本與已知漏洞
- [ ] 項次 63：委外廠商應提供源碼檢測報告及弱點掃描報告

### 構面 6：系統與通訊保護 — 新增
- [ ] 項次 70：應停用不安全的加密協定（SSL 2.0/3.0、TLS 1.0/1.1）
- [ ] 項次 71：應使用強密碼套件（Cipher Suites），避免已知弱點演算法
- [ ] 項次 74：機敏性資料應加密儲存於資料庫

### 構面 7：系統與資訊完整性 — 新增
- [ ] 項次 78：應建立自動化更新機制（WSUS / yum-cron）
