# 正規化需求規格書 (Formal Requirements)

> 專案：各公司各季度財務分析與股價預測系統
> 版本：v0.2　最後更新：2026-07-14
> 來源：`inputs/user_requirement_raw.md`

## 一、系統範圍界定

### 系統要做什麼
- 定期擷取台股與美股上市（櫃）公司之季度財務資料，正規化為統一資料模型。
- 計算並儲存 7 項核心財務指標：營收、EPS、毛利率、淨利率、負債比、現金流、本益比。
- 結合「財報因子統計模型」與「股價時間序列模型」，產出每週更新的股價預測（漲跌方向 + 幅度區間）。
- 提供 API 供下游系統或使用者查詢分析結果與預測結果。
- 追蹤預測準確率，供模型迭代與維運監控使用。

### 系統不做什麼（Out of Scope，防止 AI 生成冗餘程式碼）
- 不提供網頁儀表板 / 前端可視化介面（本期僅後端服務 + API）。
- 不做即時（Tick-level）交易資料處理，僅處理日頻股價與季頻財報。
- 不提供下單、交易執行或任何券商交易介接功能。
- 不對台股、美股以外之市場（如陸股、歐股）提供資料支援。
- 不提供投資建議之法律責任承擔（僅供分析參考，非投資建議）。

### REQ_001：台股財務資料擷取（MOPS）
- **描述**：定期自公開資訊觀測站擷取台股上市（櫃）公司之季度財報（營收、EPS、毛利率、淨利率、負債比、現金流、本益比相關原始欄位）。
- **驗收條件**：
  - 可指定股票代號清單，批次擷取最近 N 季財報。
  - 擷取失敗（來源異常、格式變更）需記錄錯誤並可重試，不可中斷整批任務。
  - 每季財報公布後 24 小時內完成擷取更新。

### REQ_002：美股財務資料擷取（SEC EDGAR + 市場資料 API）
- **描述**：自 SEC EDGAR 擷取美股上市公司 10-Q/10-K 申報資料，並自市場資料 API 取得對應股價資料。
- **驗收條件**：
  - 可指定美股代號（Ticker）清單，批次擷取最近 N 季申報資料。
  - SEC EDGAR 申報格式（XBRL）需正確解析對應至統一財務指標欄位。
  - 市場資料 API 供應商須可替換（介面抽象化），不得寫死單一供應商。

### REQ_003：跨市場財務指標正規化與儲存
- **描述**：將台股/美股原始財報資料正規化為統一資料模型，供後續模型讀取。
- **驗收條件**：
  - 統一貨幣單位標註（TWD/USD），不強制換匯，但需明確標示幣別。
  - 7 項核心指標（營收、EPS、毛利率、淨利率、負債比、現金流、本益比）不可為空值超過合理容忍範圍，缺漏需明確標記。
  - 資料版本化：同一季度資料若因財報更正而重新公布，需保留歷史版本並標記最新版。

### REQ_004：財報因子統計模型
- **描述**：以正規化後的財務指標為特徵，建立統計模型（如迴歸/分類），預測未來一週股價漲跌方向與幅度區間。
- **驗收條件**：
  - 模型輸入需包含至少 7 項核心財務指標之衍生特徵（如年增率、環比變化）。
  - 輸出格式：`{方向: 漲/跌/平, 幅度區間: [下限%, 上限%], 信心分數: 0~1}`。
  - 模型訓練與推論流程需可重現（固定隨機種子、版本化模型檔案）。

### REQ_005：股價時間序列模型
- **描述**：以歷史股價序列為輸入，捕捉短期價格動能，作為財報因子模型的補充訊號。
- **驗收條件**：
  - 至少支援一種時間序列方法（如 ARIMA / Prophet / LSTM 等，實作時擇一或多種比較）。
  - 需能處理股價資料缺漏（如停牌、假日）。
  - 輸出格式須與 REQ_004 相容，供 REQ_006 融合。

### REQ_006：雙模型融合（Ensemble）與預測結果產出
- **描述**：將財報因子模型與時間序列模型之輸出進行融合，產出最終每週預測結果。
- **驗收條件**：
  - 融合邏輯需可設定權重或可切換融合策略（如加權平均、堆疊模型）。
  - 最終輸出需保留兩個子模型的個別預測結果，供可解釋性追溯。

### REQ_007：預測結果查詢 API
- **描述**：提供 API 端點供查詢指定公司之最新財務分析摘要與股價預測結果。
- **驗收條件**：
  - 提供依股票代號查詢財務指標歷史序列之端點。
  - 提供依股票代號查詢最新一週預測結果之端點。
  - API 需具備身分驗證（對應安全需求 REQ_SEC_001）。

### REQ_008：資料擷取與模型更新排程機制
- **描述**：建立定期排程，自動觸發財報擷取（依季報公布週期）與股價/預測更新（依週）。
- **驗收條件**：
  - 排程失敗需通知（記錄於日誌，Phase 06 監控可追蹤）。
  - 排程任務可手動觸發重跑（供異常修復）。

### REQ_009：預測準確率回測與追蹤報告
- **描述**：追蹤歷史預測結果與實際股價表現之準確率，供模型迭代參考。
- **驗收條件**：
  - 提供方向準確率（Directional Accuracy）與幅度區間命中率統計。
  - 報告至少涵蓋近 12 週滾動視窗。

## 三、資料模型（概念層級，詳細 Schema 於 Phase 02 設計）

| 實體 | 說明 | 核心欄位（節錄） |
| :--- | :--- | :--- |
| `Company` | 公司主檔 | 代號、名稱、市場別（TW/US）、產業別 |
| `FinancialReport` | 季度財報正規化資料 | 公司代號、季度、營收、EPS、毛利率、淨利率、負債比、現金流、本益比、幣別、資料版本 |
| `PriceHistory` | 歷史股價資料 | 公司代號、日期、開高低收、成交量 |
| `Prediction` | 預測結果 | 公司代號、預測基準週、方向、幅度區間、信心分數、子模型明細（財報模型/時間序列模型） |
| `PredictionBacktest` | 回測記錄 | 預測 ID、實際結果、是否命中方向、是否命中區間 |

## 四、建議技術棧（待 Phase 02/03 確認）
