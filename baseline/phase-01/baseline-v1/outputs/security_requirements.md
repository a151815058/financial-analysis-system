# 安全需求規格 (Security Requirements)

> 防護等級：中級 (Medium)　適用構面（Phase 01）：構面 5（系統與服務獲得—需求階段）
> 對應參考：`external-resources/Security-Principles/references/05_acquisition.md`

## 一、CIA 等級定義

| 面向 | 等級 | 說明 |
| :--- | :--- | :--- |
| 機密性 (Confidentiality) | 中 | 財報資料多為公開資訊（MOPS/SEC EDGAR 皆公開），但預測模型、API 金鑰與系統內部資料庫存取需管控 |
| 完整性 (Integrity) | 高 | 財務資料與預測結果之正確性直接影響下游決策，需防止資料竄改與模型輸出被竄改 |
| 可用性 (Availability) | 中 | 屬分析型系統，非即時交易系統，可容忍短暫中斷，但排程任務需具備重試機制 |

## 二、威脅建模範圍（提要，詳細於 Phase 02 補齊 `threat_model.md`）

| 威脅情境 | 說明 | 初步對應措施 |
| :--- | :--- | :--- |
| 外部資料源異常/惡意資料注入 | MOPS/SEC EDGAR/市場資料 API 回傳異常或被竄改資料 | 輸入驗證、資料範圍檢查、異常值偵測（構面 7） |
| API 未授權存取 | 財務分析與預測結果被未授權第三方存取 | API 金鑰/權杖驗證、最小權限（構面 1、4） |
| 憑證/金鑰外洩 | 第三方 API 金鑰或資料庫憑證外洩 | 密鑰管理、環境變數隔離、不進版控（構面 1、6） |
| 模型/資料完整性破壞 | 訓練資料或模型檔案被竄改 | 版本化、雜湊驗證、存取控制（構面 7） |
| 傳輸過程被竊聽 | API 與外部資料源傳輸未加密 | 全面 TLS/HTTPS（構面 6） |

## 三、法規遵循清單

| 項目 | 說明 |
| :--- | :--- |
| 資料來源合規性 | 僅使用 MOPS、SEC EDGAR 等公開合法資料來源，不得違反其使用條款（如爬蟲頻率限制） |
| 免責聲明 | 系統輸出僅供分析參考，非投資建議，需於 API 回應或文件中明確聲明 |
| 個資保護 | 本系統不處理個人資料（僅公司財報與市場資料），暫不適用個資法特別規範 |

## 四、OWASP 相關防範重點（Phase 03/04 落實）

- 輸入驗證：所有外部資料源回傳內容需經過 Schema 驗證，防止注入或格式異常導致的解析錯誤。
- 敏感資訊管理：API 金鑰、資料庫憑證一律透過環境變數或密鑰管理服務，不寫入原始碼或版控。
- 相依套件安全：定期執行 SBOM 掃描（Phase 05），追蹤第三方套件已知漏洞。

## 五、適用安全構面對照（本專案，中級 medium）

| 構面 | 主要適用階段 | 本專案關聯重點 |
| :--- | :--- | :--- |
| 1. 存取控制 | Phase 02/03 | API 金鑰管理、最小權限存取財務資料庫 |
| 2. 事件日誌與可歸責性 | Phase 03/06 | 資料擷取、模型推論、API 呼叫軌跡記錄 |
| 3. 營運持續計畫 | Phase 05/06 | 資料庫備份、模型檔案備援、排程失敗重試 |
| 4. 識別與鑑別 | Phase 02/03 | API 呼叫者身份驗證機制 |
| 5. 系統與服務獲得 | Phase 01~05 | 本文件即構面 5 之需求階段落實 |
| 6. 系統與通訊保護 | Phase 02/03 | 對外部資料源與內部 API 均使用 TLS/HTTPS |
| 7. 系統與資訊完整性 | Phase 04/06 | 外部資料輸入驗證、模型/資料版本完整性、弱點掃描 |
